# gestao-financeira-api-Hexagonal-J25
API para gerenciamento de pessoas, receitas e despesas. Convenções de Commit Este projeto segue um padrão simplificado de Commits Convencionais.Essa versão usa arquitetura hexagonal e java 25 comparada a versao normal que foi feita na arquitetura layered.Uso do hexagonal é para cosneguir ter teste mais rapidos,ficar conhecimento na arquitetura e ter menos acomplamento e depencia do spring boot sendo mais facil trocar ou framework,banco de dados ou conectar uma api sem mudar/quebrar muitos arquivos.
